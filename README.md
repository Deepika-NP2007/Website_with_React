# Aksha_AI — Flask Backend

AI-powered skin disease screening API built with Flask and TensorFlow (MobileNetV2).

---

## Project Structure

```
aksha_backend/
├── app.py              ← Flask API (main file)
├── train.py            ← Model training script
├── requirements.txt    ← Python dependencies
├── uploads/            ← Temp folder for uploaded images (auto-created)
├── models/
│   └── aksha_skin_model.h5   ← Trained model (generated by train.py)
└── dataset/            ← Your training images (you provide this)
    ├── train/
    │   ├── Acne & Pimples/
    │   ├── Eczema/
    │   └── Normal Skin/ ...
    └── val/
        └── ...
```

## Setup

```bash
# 1. Create virtual environment
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Run the Flask server
python app.py
# Server starts at http://localhost:5000
```

## API Endpoints

### `GET /api/health`
Health check.
```json
{ "status": "healthy", "model_loaded": true }
```

---

### `POST /api/predict`
Predict skin condition from a face/skin image.

**Option A — File upload (multipart)**
```bash
curl -X POST http://localhost:5000/api/predict \
  -F "image=@/path/to/face.jpg"
```

**Option B — Base64 JSON**
```bash
curl -X POST http://localhost:5000/api/predict \
  -H "Content-Type: application/json" \
  -d '{"image": "data:image/jpeg;base64,/9j/4AAQ..."}'
```

**Response**
```json
{
  "status": "success",
  "primary": {
    "condition": "Acne & Pimples",
    "confidence": 87.4,
    "severity": "mild",
    "description": "Inflammatory acne...",
    "recommendation": "Use gentle, non-comedogenic skincare...",
    "color": "#d85a30"
  },
  "top3": [...],
  "all_results": [...],
  "disclaimer": "This is an AI-generated screening result and NOT a medical diagnosis..."
}
```

---

### `POST /api/predict/batch`
Predict for up to 5 images at once.
```bash
curl -X POST http://localhost:5000/api/predict/batch \
  -F "images[]=@face1.jpg" \
  -F "images[]=@face2.jpg"
```

List all 12 detectable conditions with metadata.

### `GET /api/model/info`

## Training Your Model

```bash
# Organise your dataset (see folder structure above), then:
python train.py --dataset ./dataset --epochs 20 --batch_size 32 --fine_tune
```

The script runs in **two phases**:
1. **Phase 1** — Trains only the new top layers (base MobileNetV2 frozen)
2. **Phase 2** — Fine-tunes the top 30 MobileNetV2 layers with a low learning rate

Recommended dataset: **300–500 images per class minimum** for reliable accuracy.

### Recommended public datasets
- [ISIC Skin Lesion Archive](https://www.isic-archive.com/)
- [DermNet NZ](https://dermnetnz.org/)
- [HAM10000 on Kaggle](https://www.kaggle.com/datasets/kmader/skin-lesion-analysis-toward-melanoma-detection)

## Connecting to Aksha.html Frontend

In your `Aksha.html`, inside the `startScanFlow()` function (or wherever you handle the scan), add:

```javascript
async function runAIPrediction(imageFile) {
  const formData = new FormData();
  formData.append('image', imageFile);

  const response = await fetch('http://localhost:5000/api/predict', {
    method: 'POST',
    body: formData,
  });
  const result = await response.json();

  if (result.status === 'success') {
    // Show result.primary.condition, result.primary.confidence, etc.
    displayResults(result);
  }
}

function displayResults(result) {
  const { condition, confidence, severity, description, recommendation } = result.primary;
  // Update your results page DOM elements here
  document.getElementById('diagnosis-title').textContent = condition;
  document.getElementById('confidence-score').textContent = `${confidence}%`;
  // ... etc
}
```

For **base64 from webcam**:
```javascript
async function scanFromCanvas(canvas) {
  const base64 = canvas.toDataURL('image/jpeg');
  const response = await fetch('http://localhost:5000/api/predict', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ image: base64 }),
  });
  return await response.json();
}
```

## Notes
- The model is **not a medical device**. Always show the disclaimer to users.
- Image size limit is **10 MB** (configurable in `app.config`).
- Supported formats: JPG, PNG, WEBP.
- The `uploads/` folder stores images temporarily — add a cleanup job in production.
